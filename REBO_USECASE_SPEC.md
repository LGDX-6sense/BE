# REBO 유스케이스 및 기능 명세

이 문서는 현재 저장소 구현 상태를 기준으로 REBO 기능을 다시 정리한 버전입니다.
기획 초안과 실제 코드 사이 차이가 있는 항목은 `구현`, `부분 구현`, `미구현`으로 구분했습니다.

## 1. 문서 범위

- 기준 저장소: `BE copy/`
- 대상 구성: Flutter 모바일 앱 + FastAPI 백엔드 + 멀티모달 AI 에이전트
- 반영 기준: 2026-03-24 현재 코드 구현

## 2. 유스케이스

### UC-R-001 고장 증상 문의

- 관련 요구사항: `B001`
- 사용자: 고객
- 선행 조건: REBO 챗봇 화면 또는 ChatThinQ 내 REBO 진입 화면에 접속한 상태

기본 흐름

1. 고객이 REBO 챗봇에서 고장 증상을 문의한다.
2. 시스템이 다음 입력 방법을 제공한다. 중복 선택 가능하다.
3. `사진 첨부` 선택 시 `UC-R-002`를 수행한다.
4. `음성 녹음` 선택 시 `UC-R-003`을 수행한다.
5. `텍스트 입력` 선택 시 `UC-R-004`를 수행한다.
6. 입력이 준비되면 `UC-R-005 접수 내용 통합 분석 및 솔루션 제시`를 수행한다.
7. REBO가 분석 결과, 자가점검 안내 또는 서비스 전환 안내를 고객에게 표시한다.

예외

- 입력 없이 이탈하면 챗봇 홈으로 복귀한다.

종료 조건

- 고객 문의에 대한 AI 분석 결과 또는 다음 단계 안내가 제공된다.

우선순위

- 서비스 중요도 `상`

### UC-R-002 사진 첨부

- 관련 요구사항: `B002`, `C001`
- 사용자: 고객
- 선행 조건: `UC-R-001`이 시작된 상태

기본 흐름

1. 고객이 `사진 첨부`를 선택한다.
2. 시스템이 카메라 촬영 또는 갤러리 선택 옵션을 제공한다.
3. 고객이 제품 상태 화면 또는 오류 화면 이미지를 선택한다.
4. 시스템이 선택된 이미지를 미리보기로 표시한다.
5. 고객은 추가로 텍스트, 오디오 파일, 음성 녹음을 병행할 수 있다.
6. REBO AI가 이미지를 분석하고 `UC-R-005`로 전달한다.

예외

- 카메라 또는 갤러리 권한이 거부되면 다른 입력 방식 사용을 유도한다.
- 이미지 선택이 실패하면 재선택을 요청한다.

종료 조건

- 이미지가 첨부되고 AI 분석 대기 상태가 된다.

현재 구현 메모

- 현재 앱은 사진을 `1장`만 첨부한다. 초안의 `최대 3장`은 아직 미구현이다.

### UC-R-003 음성 녹음

- 관련 요구사항: `B003`, `C002`, `C003`
- 사용자: 고객
- 선행 조건: `UC-R-001`이 시작된 상태

기본 흐름

1. 고객이 MIC 버튼을 클릭한다.
2. 시스템이 녹음 방식 선택 팝업을 표시한다.
3. 고객은 `음성 채팅` 또는 `소음 녹음` 중 하나를 선택한다.
4. 음성 채팅 선택 시 문제 상황을 음성으로 설명한다.
5. 소음 녹음 선택 시 제품의 이상 소리를 녹음한다.
6. 녹음 완료 후 시스템이 오디오 파일을 생성한다.
7. 음성 채팅은 STT 변환 후 AI 분석으로 전달된다.
8. 소음 녹음은 오디오 분석 입력으로 AI 분석에 전달된다.

예외

- 마이크 권한이 거부되면 텍스트 입력을 유도한다.
- 녹음 실패 시 재녹음 또는 다른 입력 방식을 안내한다.

종료 조건

- 녹음 데이터가 저장되고 AI 분석 대기 상태가 된다.

현재 구현 메모

- 음성 채팅과 소음 녹음이 분리되어 있다.
- 앱은 별도 오디오 파일 첨부도 지원한다.

### UC-R-004 텍스트 입력

- 관련 요구사항: `B004`
- 사용자: 고객
- 선행 조건: `UC-R-001`이 시작된 상태

기본 흐름

1. 고객이 채팅 입력창에 제품명, 오류 코드, 증상 등을 입력한다.
2. 입력값 또는 첨부 데이터가 존재하면 전송 버튼이 활성화된다.
3. 고객은 사진, 오디오 파일, 음성 녹음을 추가로 병행할 수 있다.
4. REBO AI가 텍스트를 분석하여 `UC-R-005`로 전달한다.

예외

- 텍스트와 첨부가 모두 비어 있으면 전송되지 않는다.

종료 조건

- 텍스트가 전송되고 AI 분석 대기 상태가 된다.

현재 구현 메모

- 초기 상태에서는 MIC 중심 UI를 사용하고, 텍스트나 첨부가 있으면 전송 동작이 활성화된다.

### UC-R-005 접수 내용 통합 분석 및 솔루션 제시

- 관련 요구사항: `C001`, `C002`, `C003`, `C004`, `C005`, `C006`
- 사용자: REBO AI Agent
- 선행 조건: `UC-R-001`에서 입력 데이터가 준비된 상태

기본 흐름

1. REBO가 사진, 텍스트, 음성, 소음 데이터를 통합 분석한다.
2. REBO가 LG 제품 지식베이스와 유사 증상 검색 결과를 참고해 원인을 추론한다.
3. 자가 해결 가능 여부를 판단한다.
4. 자가 해결 가능 시 단계별 자가점검 또는 자가조치 방법을 안내한다.
5. 자가 해결이 어렵거나 사용자가 서비스 요청 의사를 보이면 `UC-R-007`로 전환한다.
6. 응답은 채팅창에서 순차 표시 방식으로 보여 준다.

예외

- 지식베이스 검색 실패 시 기본 안내와 서비스 전환 안내를 제공한다.
- 분석 정보가 부족하면 추가 정보 입력을 요청한다.

종료 조건

- 분석 결과, 자가조치 또는 서비스 전환 안내가 고객에게 표시된다.

현재 구현 메모

- 유사 증상 검색은 Pinecone 우선, 필요 시 Supabase 또는 로컬 데이터 fallback 구조를 사용한다.
- 현재 앱의 응답 표시 애니메이션은 `가짜 스트리밍 UI`에 가깝고, 실제 서버 스트리밍 취소는 지원하지 않는다.

### UC-R-006 대화 보관함 조회 및 재진입

- 관련 요구사항: `D005`
- 사용자: 고객
- 선행 조건: 이전 대화가 저장된 상태

기본 흐름

1. 고객이 보관함 화면을 연다.
2. 시스템이 세션 제목, 요약, 일시를 포함한 목록을 표시한다.
3. 고객이 특정 대화를 선택한다.
4. 시스템이 해당 세션의 대화 상세를 불러온다.
5. 고객은 이전 대화 내용을 확인하거나 해당 세션으로 재진입한다.

예외

- 보관함 로딩 실패 시 재시도 UI를 제공한다.

종료 조건

- 고객이 과거 대화를 확인하거나 이어서 사용할 수 있다.

현재 구현 메모

- 현재 구현은 `고객용 보관함` 중심이다.
- 초안의 `기사/상담사 전용 진단 요약 보고서 화면`은 아직 별도 UI로 구현되지 않았다.

### UC-R-007 서비스 전환 안내

- 관련 요구사항: `C005`
- 사용자: 고객, REBO AI Agent
- 선행 조건: `UC-R-005`에서 자가 해결이 어렵거나 고객이 서비스 요청을 직접 입력한 상태

기본 흐름

1. AI가 서비스 전환이 필요하다고 판단하거나 고객이 `AS 신청`, `상담 서비스`, `출장 서비스`를 요청한다.
2. 시스템이 다음 세 가지 선택지를 표시한다.
3. `자가점검` 선택 시 자가점검 안내를 제공한다.
4. `상담 서비스` 선택 시 `UC-R-008`을 수행한다.
5. `출장 서비스` 선택 시 `UC-R-009`를 수행한다.

예외

- 사용자가 선택을 취소하면 현재 대화 화면에 머문다.

종료 조건

- 고객이 다음 단계 서비스 경로를 선택한다.

현재 구현 메모

- 이전 2분기 구조가 아니라 현재는 `자가점검 / 상담 서비스 / 출장 서비스` 3분기 구조다.

### UC-R-008 상담 서비스 예약 안내

- 관련 요구사항: `C005`, `D001`
- 사용자: 고객
- 선행 조건: `UC-R-007`에서 `상담 서비스`를 선택한 상태

기본 흐름

1. 시스템이 `상담 서비스 예약을 도와드릴게요!` 안내 카드를 표시한다.
2. 시스템이 `상담 서비스 예약` 버튼을 제공한다.
3. 고객이 버튼을 눌러 상담 서비스 예약 단계로 이동한다.

예외

- 버튼 클릭 후 추가 정보가 부족하면 제품 정보 및 증상 입력을 다시 요청할 수 있다.

종료 조건

- 고객이 상담 서비스 예약 흐름으로 진입한다.

현재 구현 메모

- 현재 저장소는 `예약 안내 카드와 버튼`까지 구현되어 있다.
- 실제 개인정보 입력 폼, 상담사 배정, 예약 확정 화면은 아직 별도 구현이 필요하다.

### UC-R-009 출장 서비스 안내

- 관련 요구사항: `C005`, `D001`
- 사용자: 고객
- 선행 조건: `UC-R-007`에서 `출장 서비스`를 선택한 상태

기본 흐름

1. 시스템이 출장 서비스 안내 카드를 표시한다.
2. 시스템이 출장 서비스 진행 버튼을 제공한다.
3. 고객이 버튼을 눌러 출장 서비스 접수 흐름으로 이동한다.

예외

- 추가 정보가 부족하면 제품 정보 또는 증상 입력을 다시 요청할 수 있다.

종료 조건

- 고객이 출장 서비스 접수 흐름으로 진입한다.

현재 구현 메모

- 현재 저장소는 `출장 서비스 안내 카드` 중심 구현이다.
- 희망 방문 일시 선택, 기사 배정, 상태 조회는 아직 별도 구현이 필요하다.

### UC-R-010 AI 리포트 생성 및 조회

- 관련 요구사항: `D005`, `E001`
- 사용자: 고객, 상담사, 수리기사
- 선행 조건: `UC-R-005`가 완료되었거나 `UC-R-007` 이후 서비스 전환 이력이 저장된 상태

기본 흐름

1. 고객의 대화가 저장될 때 시스템이 세션 제목과 AI 요약을 자동 생성한다.
2. 시스템은 대화 세션 상태, 요약 문장, 전체 대화 이력을 함께 보관한다.
3. 고객이 보관함에서 해당 세션을 열면 AI 요약과 상세 대화를 확인할 수 있다.
4. 상담사 또는 수리기사는 저장된 대화와 요약 정보를 참고해 후속 응대를 준비할 수 있다.

예외

- 요약 생성에 실패하면 기본 제목과 기본 안내 문구로 저장한다.
- 저장 실패 시 보관함 반영이 누락될 수 있으며, 앱에는 경고 문구가 표시될 수 있다.

종료 조건

- 대화 세션에 대한 AI 리포트가 저장되고, 이후 조회 가능한 상태가 된다.

현재 구현 메모

- 현재 AI 리포트는 별도 전용 화면이 아니라 `보관함 카드 제목/요약 + 세션 상세 대화` 형태로 제공된다.
- 상담사/기사 전용 보고서 UI는 아직 없지만, 현재 저장 데이터는 후속 보고서 UI의 기반으로 사용할 수 있다.

## 4. AI 리포트 정의

현재 저장소 기준 AI 리포트는 아래 정보를 중심으로 구성됩니다.

- 리포트 제목: 대화 핵심을 짧게 요약한 세션 제목
- AI 요약: 증상, 진단, 조치, 해결 여부, 서비스 상태를 1~2문장으로 정리한 문장
- 세션 상태: 진행 중, 해결됨, 보관됨 등 보관함 상태값
- 접수 이력: 고객 발화와 AI 응답 전체 대화
- 서비스 전환 이력: 자가점검, 상담 서비스, 출장 서비스, AS 유도 여부
- 참고 근거: AI 응답 생성 시 수집된 evidence 및 검색 컨텍스트

현재 구현 범위

- 백엔드가 대화 저장 시 자동으로 제목과 요약을 생성한다.
- 앱 보관함 카드에서 `AI 요약` 라벨로 요약 문장을 보여 준다.
- 세션 상세 화면에서는 전체 대화 기록을 다시 확인할 수 있다.

후속 확장 권장 항목

- 상담사/수리기사용 전용 AI 리포트 화면
- 접수 방법 구분 표시: 텍스트, 사진, 음성 채팅, 소음 녹음
- 자가점검 시도 여부 및 결과 구조화
- 고객 제품 정보, 모델명, 희망 방문 일시 등 구조화 필드 추가

## 5. 기존 초안 대비 핵심 수정 사항

- 사진 첨부는 현재 `최대 3장`이 아니라 `1장` 기준이다.
- 서비스 전환은 `상담원 연결/출장 예약` 2분기가 아니라 `자가점검 / 상담 서비스 / 출장 서비스` 3분기다.
- 상담 서비스 선택 시 `상담 서비스 예약 안내 카드`와 버튼이 표시된다.
- 기사/상담사용 전용 `진단 내용 요약 보고서 화면`은 아직 없고, 현재는 고객용 `보관함 요약`이 해당 역할 일부를 대체한다.
- `응답 생성 중단`, `실시간 스트리밍`, `실제 A/S 예약 확정`은 아직 후속 구현 항목이다.
